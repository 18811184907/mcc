# MCC v2.6.1 · 上下文优化

## 起源

v2.5.x + v2.6.0 累积下来 MCC baseline 占 200k context 的 ~7-8%（约 15k token）。这次做两件事降占用，不动功能。

## 改动

### #1 收紧 6 个过长 description（每 session baseline 减 ~720 字符）

prompt-engineer agent 扫了 23 skill + 19 agent 的 frontmatter，>300 字符的有 6 个。压缩后保留**触发场景**和**与相邻 skill/agent 的分工标识**，删冗余强调和实现细节。

| 文件 | Before | After | 节省 |
|---|---:|---:|---:|
| user-vault | 506 | 248 | -258 |
| project-vault | 377 | 240 | -137 |
| claudemd-sync | 340 | 218 | -122 |
| dev-changelog | 338 | 235 | -103 |
| database-schema-doc | 333 | 232 | -101 |
| silent-failure-hunter (agent) | 320 | 248 | -72 |
| **合计** | | | **-793** |

每个压缩版自查 checklist：
- ✓ 触发场景仍在前
- ✓ 触发例句保留（具体到用户原话级）
- ✓ 与相邻 skill/agent 的分工标识保留或新增（防误激活）
- ✓ 删的只是 emphasis 标记、营销语、实现细节

### #2 rules/typescript paths 按需触发（编辑 .ts 时减 ~8.5k 字符）

code-explorer agent 调查了 `~/.claude/rules/typescript/*.md` 的注入机制：是 Claude Code harness 内置功能，按 frontmatter `paths:` glob 匹配文件路径，每次匹配的 tool result 后注入全文。

之前 5 个文件都挂在 `**/*.ts/.tsx/.js/.jsx` 这一级，**改任何 .ts 文件都注入 5 个全文**（合计 ~33k 字符）。

收窄两个最不该全量加载的：

**testing.md (~3.8k chars)** — 之前每次 .ts 编辑都注入 → 现在只在改测试文件时：
```yaml
paths:
  - "**/*.test.{ts,tsx,js,jsx}"
  - "**/*.spec.{ts,tsx,js,jsx}"
  - "**/test/**/*.{ts,tsx,js,jsx}"
  - "**/tests/**/*.{ts,tsx,js,jsx}"
  - "**/__tests__/**/*.{ts,tsx,js,jsx}"
  - "**/playwright/**/*.{ts,tsx,js,jsx}"
  - "**/vitest.config.*"
  - "**/jest.config.*"
  - "**/playwright.config.*"
```

**hooks.md (~4.7k chars)** — 之前每次 .ts 编辑都注入 → 现在只在改 hook/工具配置时：
```yaml
paths:
  - "**/.claude/settings*.json"
  - "**/.claude/hooks/**"
  - "**/.claude/.mcc-hooks/**"
  - "**/.husky/**"
  - "**/lefthook.{yml,yaml}"
  - "**/package.json"
  - "**/.eslintrc*"
  - "**/eslint.config.*"
  - "**/.prettierrc*"
  - "**/prettier.config.*"
  - "**/tsconfig*.json"
```

保持宽 paths 的：
- `coding-style.md` — 几乎所有 TS/JS 编辑都需要（基础规范）
- `patterns.md` — 已经只 .ts/.tsx
- `security.md` — 安全适用面广，保留
