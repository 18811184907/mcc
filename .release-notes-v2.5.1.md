# MCC v2.5.1 · AI 主导接管模式 + database-schema-doc skill

## 起源：用户反馈 v2.5.0 流程太复杂

v2.5.0 文档让用户：
1. 拷模板
2. 编辑填值
3. 保存等 hook 跑

用户说："**应该是项目开发过程中 AI 把相关内容自动写到 PROJECT_VAULT.md，每个项目各有一个**"。

跟着用户继续提了元偏好："**SCHEMA.md 也应该 AI 自动写入。给全局 CLAUDE 增加偏好：AI 开发，大多数要求自动写入而不是人为操作**"。

v2.5.1 全部体现这个哲学。

## 三件事

### 1. project-vault skill 重写为"AI 主动接管"

旧（v2.5.0）：用户 cp 模板 → 编辑 → 保存。
**新（v2.5.1）**：用户在对话提"OPENAI_API_KEY 是 sk-xxx" / "服务器 192.168.1.10 用户 deploy" → Claude 自动：
1. 检查 `<cwd>/.claude/PROJECT_VAULT.md` 是否存在
2. 不存在 → 自动从模板拷 + 删占位 → 保留章节标题
3. 存在 → Edit 加新条目
4. PostToolUse hook 触发同步到 .env.local / .env.example / ~/.ssh/config / SECRETS-INDEX.md
5. 简短报告："✓ 已加 OPENAI_API_KEY 到 vault"

触发场景大幅扩充：
- A: 用户直接说出 secret 值
- B: Claude 在工作中产出（生成 JWT secret / 配 OAuth client_secret / webhook HMAC）
- C: 防御场景（用户硬编 secret → pre-secret-detect hook 警告 → Claude 主动 refactor）

### 2. NEW skill: database-schema-doc

跟 vault 一样的 AI 接管模式，针对**数据库 schema 文档**：

| 用户说 | Claude 自动做 |
|---|---|
| "加 users 表，字段 email/password" | Edit `docs/SCHEMA.md` 追加 users 段 + 字段表 |
| "users 加 deleted_at 软删字段" | Edit users 段补字段 |
| "users.email 加唯一索引" | 更新索引段 |
| "decision: 用 Postgres 不用 MySQL" | 写 `docs/adr/0001-*.md`（不塞 SCHEMA.md） |

ORM 自动 stub 规则：
- 检测到 `prisma/schema.prisma` → 解析 model 块 → stub 表骨架
- 检测到 `alembic/versions/*.py` → 解析 → stub
- 检测到 `db/schema.rb` (Rails) → 解析 → stub
- 业务含义优先级：对话上下文 > 字段命名 > 注释 > `_TODO_` 占位

`docs/SCHEMA.md` **进 git**（团队 onboarding 必读）。

新模板：`source/templates/SCHEMA.example.md`，含 users / orders / sessions 三张表的真实业务示例。

### 3. CLAUDE.global.example.md 加 "AI 主导开发" 偏好段

最高优先级偏好。Claude 看到这段必须改变行为：

| 原则 | 内容 |
|---|---|
| **自动接管** | 凡 MCC 提供 skill / hook 能自动化的，Claude 必须主动接管，不要先问 |
| **6 类典型场景对应 skill** | secret → vault；DB schema → schema-doc；决策 → ADR；bug → mistakes；架构改 → PRP；项目事实 → project memory |
| **禁止行为** | 让用户拷模板 / 手编 .env/.ssh / 先问要不要 / 把决策推回给用户 |
| **例外（必须先问）** | 改 git 远程 / 发出去操作 / 不可逆破坏 / 超范围重构 |
| **报告格式** | "✓ 已写入 X / 改动一句 / 撤回方法" 简短一句 |

## 升级

```powershell
iwr -useb https://raw.githubusercontent.com/18811184907/mcc/main/bootstrap.ps1 | iex
```

老用户重跑会装上新 skill。**但你本机的 `~/.claude/CLAUDE.md` 不会被自动覆盖**（installer 检测到存在就跳过 —— 设计原则：用户级 CLAUDE.md 由用户掌控）。

要让"AI 主导开发"段在你本机也生效，让 Claude 帮你合并：

> "把 ~/.mcc-install/source/templates/CLAUDE.global.example.md 里的 'AI 主导开发' 段合并到我本机 ~/.claude/CLAUDE.md"

Claude 会自动 Read 两边 + Edit 你的本机文件。
