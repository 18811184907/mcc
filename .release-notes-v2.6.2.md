# MCC v2.6.2 · 第二轮 opus 审计 C 档完整修复

## 起源

v2.6.0 加 USER_VAULT 之后做了第二轮 opus 模型审计——4 个 agent 并行跑 security / code-review / silent-failure / javascript-pro。这次修了 13 条跨 agent 标红的问题 + 抽 `lib/vault-parser.js` 消除 DRY。

## 主轴：抽共享 `lib/vault-parser.js`

之前 `parseVault` / `sanitizeSshField` / `expandHome` / `escapeRegex` 在 3 个 hook 文件里各有一份，行为已经分歧——`post-vault-sync.js` 缺 placeholder 过滤，会把 `<your-key>` 模板占位写到 `.env.local`。**这是真 bug，被 DRY 违反掩盖了。**

新 `lib/vault-parser.js` 单一权威：
- `parseVault(content)` —— 解析 vault markdown 含 placeholder 过滤
- `isPlaceholder(value)` —— 识别 `<...>` / `your-...` / `placeholder` / `todo` / `xxx`
- `sanitizeSshField(name, value)` —— 拒绝 `\r\n\0`
- `expandHome(p)` —— `~/` 展开
- **`secureWrite(path, content, opts)` —— symlink-safe + atomic tmp+rename + Windows ACL lockdown**
- **`appendOnce(path, block, marker, opts)` —— O_EXCL 锁 + tmp+rename 真原子幂等**

## 安全 / 数据完整性修复（高 ROI）

### 🔒 secureWrite 替换所有 vault 写点

**之前**：`fs.writeFileSync(path, content, { mode: 0o600 })`
- ❌ 不检查 path 是否 symlink → 恶意 npm 包可让 `~/.claude/.user-env.sh` symlink 到 `~/.bashrc`，MCC 写入会**覆盖** .bashrc
- ❌ 非原子 → watchdog 5s 杀进程时可能留半写文件
- ❌ Windows 上 `mode: 0o600` silently ignored → 文件用默认 ACL（同机其他用户可读）

**v2.6.2 secureWrite**：
1. `lstatSync` 检查目标，是 symlink 直接 throw
2. 写到 `<path>.mcc-tmp.<pid>` → atomic rename
3. Windows 上额外跑 `icacls /inheritance:r /grant:r <user>:F` 锁 ACL

应用到：`.user-env.sh` / `.user-env.ps1` / `~/.ssh/config` / `.env.local` / `.env.example`。

### 🔒 appendOnce 真原子化

**之前**：read marker → if absent → appendFileSync。两个 hook 进程同时跑会**双 append**（marker 重复）或半写。

**v2.6.2**：
1. `fs.openSync('<path>.mcc-lock', 'wx')` —— O_EXCL 锁，并发拿不到锁直接返回 `'locked'`
2. 写整个新内容到 tmp，atomic rename
3. 加 `skipMissing: true` 选项 —— `.bashrc`/`.zshrc` 不存在时不主动创建（fish/macOS-zsh 用户家目录不再凭空多空文件）

### 🔒 previewValue 不再泄漏 secret 字节

**之前**：`sk-***ab`（前 3 + 后 2 字节）→ stderr → Claude Code transcript → 上传给 Anthropic 推理 / 协作工具。

**v2.6.2**：`<short:N>` / `<med:N>` / `<long:N>` / `<xlong:N>` —— 只暴露长度档位，不泄漏任何字符。同时 leak 报告标 `[USER]` / `[PROJECT]` 让用户知道值出自哪边。

### 🔒 IdentityFile HOME-restriction

**之前**：只挡 `..` 路径穿越 → `/etc/shadow` 这种绝对路径能写进 `~/.ssh/config`。
**v2.6.2**：`path.resolve` 后 `startsWith(home + path.sep)` 强制 IdentityFile 必须落在 HOME 内。

## 行为修复

### Windows 同时写 PS 5.1 + PS 7+ profile（4 个 agent 全报）

之前只写 PS 7+ 路径（`Documents/PowerShell/`），但 Windows 10/11 系统自带的是 **PS 5.1**（`Documents/WindowsPowerShell/`）。用户保存 USER_VAULT，hook 跑得很欢，但 PS 5.1 shell 里 `echo $env:OPENAI_API_KEY` 永远是空。

v2.6.2：Windows 平台同时写两份 profile，PS 5.1 和 PS 7+ 用户都能拿到。

### Partial failure 不再打 ✓

之前 5 步 sync 全失败也会显示 `✓ Synced 3 user env + 2 SSH hosts`，用户被误导。
v2.6.2：partial failure 时打：

```
[user-vault-sync] ✗ 2 sync step(s) FAILED:
  - git: git config user.email failed (ENOENT): spawn git ENOENT
  - shell-autoload:bashrc: refusing to write through symlink
[user-vault-sync] partial sync — re-edit USER_VAULT.md to retry
```

不再打 ✓ Synced 假象。

### syncGitIdentity transactional rollback

之前：user.name 写成功 + user.email 失败 → 脏状态（用户改 commit 看到名字变了但邮箱没换）。
v2.6.2：先备份当前值，写失败时回滚已写入的字段。

### NUL byte 入口 reject

`shellQuote` / `psQuote` 入口检测 `\0`，遇到直接 throw（防 bash 静默丢字段）。

## v2.5.12 风格回归补全

第一轮审计后引入的 `SPAWN_OPTS` 模式（`stdio[0]:'ignore'` + `windowsHide` + `maxBuffer`）在 v2.6.0 新代码部分回归。补齐：

- `post-claudemd-sync.js` git wrapper —— 加 stdio + maxBuffer 16MB
- `auto-tmux-dev.js` `which tmux` —— 加 windowsHide + stdio + timeout
- `session-start-bootstrap.js` —— 加 windowsHide
- `run-with-flags.js` —— 加 windowsHide

Windows 用户每次 SessionStart / hook 调用不再闪 cmd 窗口。

## 老债清理

### renameAlias rollback 双失败诚实化

第一轮审计就标的 bug，v2.5.11 / v2.5.12 都没修，**v2.6.2 终于修了**。

之前：`saveAliases` 第一次失败 → rollback → `saveAliases` 第二次也失败但返回值被丢，仍然报 "已回滚"。
v2.6.2：检查 rollback 返回值，失败时报 "aliases file may be inconsistent on disk; check the .bak"。

## 新增工具 / 行为

### 空 vault 静默

只有 placeholder 的模板状态不再打 `✓ Synced 0 user env`。

### MCC_NO_AUTOLOAD=1 opt-out

环境变量设这个值后，hook 不会 wire `~/.bashrc` / `~/.zshrc` / `$PROFILE`。给用密码管理器（1Password / Bitwarden）跨机同步的高级用户。

### Watchdog timeout 不再静默

之前 5 秒 watchdog 触发直接 exit 0，用户感知"hook 跑过了但什么都没改"。v2.6.2 打 stderr 警告。

## 升级

```bash
cd <wherever-you-cloned-mcc>
git pull
node scripts/installer.js --force
```

⚠️ 升级后第一次保存 `~/.claude/USER_VAULT.md` 时，会清掉旧 `.user-env.sh` / `.user-env.ps1` 然后用 secureWrite 重新写——内容不会变，但 Windows 上会触发 `icacls` 锁 ACL（首次会有 1-2 秒延迟）。

如果之前你的 `~/.bashrc` 有重复的 MCC marker 块（v2.6.0/2.6.1 在多 session 并发下可能产生的），需要手动清一下：

```bash
# 看 bashrc 里 marker 出现几次
grep -c "# >>> MCC user-env autoload >>>" ~/.bashrc

# 如果 >1，编辑 bashrc 留一份
```
