# MCC v2.6.0 · 用户级 vault（跨项目通用 secret/SSH/git 身份）

## 起源

v2.5.x 只有项目级 `docs/PROJECT_VAULT.md`——每个项目要重复填 `OPENAI_API_KEY`、git identity、bastion SSH host 等"机器上所有项目都用"的东西。

v2.6.0 加 **用户级 vault**：`~/.claude/USER_VAULT.md`，写一次，所有项目共用。

## 三个核心组件

### 1. 新 skill：`user-vault`

Claude **自动判断**该写 USER 还是 PROJECT：

| 用户说 | Claude 写哪 |
|---|---|
| "我的 OPENAI_API_KEY 是 sk-xxx" | `~/.claude/USER_VAULT.md` |
| "git 用户名 anouk、邮箱 anouk@..." | `~/.claude/USER_VAULT.md` |
| "我的个人 GitHub PAT" | `~/.claude/USER_VAULT.md` |
| "**本项目**的 DATABASE_URL" | `<project>/docs/PROJECT_VAULT.md` |
| "Stripe live key（这个项目）" | `<project>/docs/PROJECT_VAULT.md` |

不确定时 Claude 会问。**默认 USER**（更广泛，更安全）。

### 2. 新 hook：`post-user-vault-sync.js`

USER_VAULT.md 一改完自动同步到 4 个目标：

```
~/.claude/USER_VAULT.md  →  ~/.claude/.user-env.sh    (Bash/Zsh)
                         →  ~/.claude/.user-env.ps1   (PowerShell)
                         →  git config --global       (GIT_USER_NAME/EMAIL)
                         →  ~/.ssh/config             (# MCC-User-Managed 块)
```

**第一次保存还会自动追加** `source` 行到：
- `~/.bashrc` / `~/.zshrc`
- PowerShell `$PROFILE`（PS 7+ 路径）

带 marker `# >>> MCC user-env autoload >>>` 防重复，幂等。

### 3. 双向提升

`project-vault` skill 检测到明显跨项目条目（OPENAI / ANTHROPIC / GROQ key、PERSONAL_/MY_ 前缀、bastion）时，**主动建议**：

> "OPENAI_API_KEY 看起来你别的项目也会用——要不要提到 ~/.claude/USER_VAULT.md？这样所有项目都能用，不用每个项目都写一遍。"

用户说"挪到 USER_VAULT" → Claude 从 PROJECT 删 → 加到 USER。反向也可以。

## 使用流程

**第一次（机器初装 v2.6.0+）**：

1. `node scripts/installer.js`（v2.6 起会自动建空 `~/.claude/USER_VAULT.md`）
2. 在对话里告诉 Claude："我的 OPENAI_API_KEY 是 sk-..."
3. Claude 自动写 USER_VAULT
4. hook 自动 sync + 追加 bashrc source 行
5. **当前 shell** 立刻生效手动跑一次 `source ~/.claude/.user-env.sh`；新开 shell 自动加载

**之后任何项目代码**用 `process.env.OPENAI_API_KEY` 都能拿到，不用每个项目 .env 重复写。

## 安全约束

USER_VAULT 跟 PROJECT_VAULT 同级安全规则：

- ✓ Claude 永远不在对话里复述完整 secret 值
- ✓ 不读 USER_VAULT 内容给用户看（用户自己 cat）
- ✓ pre-vault-leak-detect.js v2.6 起**同时扫 USER + PROJECT 值**——无论从哪个 vault 来的 secret，泄漏到 tool input 都告警
- ✓ SSH 字段统一走 `\r\n\0` 净化（同 v2.5.11 PROJECT_VAULT 的修复）
- ✓ Host name 严格白名单 `[A-Za-z0-9_.\-*?]`
- ✓ Port 必须是 1-99999 整数
- ✓ IdentityFile 路径不允许 `..` 穿越
- ✓ GIT_USER_* 值含换行直接拒绝（防 git config 文件破坏）

## 为什么 USER_VAULT 不进 git？

`~/.claude/` 在用户家目录，**不属于任何 git repo**——`git init` 只到当前目录的 `.git`，家目录没有 .git 自然也不被任何项目追踪。所以 USER_VAULT 永远不会 commit 到任何项目仓库，无需 .gitignore。

如果你想跨设备同步 USER_VAULT，**不要 push 到 git**——用密码管理器（1Password CLI / Bitwarden / pass / gopass）保存 secret 值，新机让 Claude 重写一份。或用 `claudemd-sync` 同款 dotfiles repo 模式自己实现（但 secret 进 git 你得自己接受风险）。

## 与 dotfiles / claudemd-sync 的区别

- `claudemd-sync` 同步 `~/.claude/CLAUDE.md`（**指令偏好**，进 dotfiles git）
- `user-vault` 同步 `~/.claude/USER_VAULT.md`（**secret 值**，**永不进 git**）

两者互补：偏好可以分享，secret 不能。

## 升级

```bash
cd <wherever-you-cloned-mcc>
git pull
node scripts/installer.js
```

升级后第一次：
1. 看 `~/.claude/USER_VAULT.md` 已经在了（installer 自动建的空模板）
2. 在对话里告诉 Claude 你的 personal API keys 和 git identity
3. Claude 自动写入 + hook 自动 sync

- USER_VAULT 加密（落盘明文，跟 PROJECT_VAULT 同模型；机器被攻陷无解）
