# MCC v2.5.12 · 审计 P1 收尾（14 条工程质量改善）

## 起源：续 v2.5.11

v2.5.11 修了 10 个 P0 Critical（4 安全 + 3 数据安全 + 3 静默失败）。
v2.5.12 把同份审计报告里的 P1 拉了 14 条工程质量改善打包出。

不修的还在 audit 里：installer 全同步 fs（要 ~1000 行重构）、runCommand → spawnSync 大改，留 v2.5.13+。

## 修复清单（按域分组）

### Hook Watchdog（4 条）

`block-no-verify.js` / `pre-bash-commit-quality.js` / `pre-bash-tmux-reminder.js` / `auto-tmux-dev.js` —— 之前一组 hook 没有 stdin watchdog，父进程异常或 stdin 不 close 时会永久挂起直到外层 hook timeout 杀掉。统一加 3-5 秒 `setTimeout` + `unref()`。

### spawnSync 工程质量统一（3 条）

| 文件 | 改动 |
|---|---|
| `stop-format-typecheck.js` | `stdio: ['ignore', 'pipe', 'pipe']` 防 tsc 挂等 stdin / `maxBuffer: 16MB` 防 tsc 大输出截断 / `windowsHide: true` / formatter 失败改 stderr 报错（之前 `catch{}` 完全吞掉） |
| `pre-bash-commit-quality.js` | 全部 spawnSync 走统一 `SPAWN_OPTS`（ignore stdin / windowsHide / maxBuffer 16MB）；`git show` 加 `--` 防文件名以 `-` 起头被解析为 option |
| `lib/utils.js` | `runCommand` + `commandExists` 加 `windowsHide` + `maxBuffer 16MB`，Windows 下不再每次 hook 闪 cmd 窗口 |

### Installer 工程改善（4 条）

- **Node 版本校验**：启动早期断言 Node ≥ 16，不再让用户撞 SyntaxError 后 ¯\\\_(ツ)\_/¯
- **dry-run 日志真诚化**：exclusive 备份的 `log('ok', ...)` 在 dry-run 下改 `(dry-run)` 前缀，不再撒谎
- **TOML array-of-tables 显式 reject**：`extractTomlSections` 遇 `[[xx]]` 抛错而非静默错位，防 appendTomlFragment 写坏 config.toml
- **.gitignore 幂等**：用 `MCC_MARKER` 字符串检查，多次重装不再重复追加（之前在用户用 `**/*.md` 通配符场景每次都加）

### Adapter 清理（3 条）

- `_lib.js parseFrontmatter`: 删 dead `currentKey` 变量 + 加注释明示 multi-line YAML 不支持
- `adapt-to-codex.js`: `unmappedWarnings` 从模块级移入函数体，重复调用不再累积旧 warning
- `adapt-to-claude-code.js copySimpleDir`: 显式取 `manifest.distDir`，避免未来传 distDir 参数时路径计算错乱
- `build.js`: agentCount / cmdCount 改用递归 `walkMd`，未来 agents/ 分子目录不再静默低估失败

### Hook 语义改善（1 条）

`bash-hook-dispatcher.js`: hook 抛异常时改写 stderr 为 "treating as allow"（之前是 "failed"，没说语义），MCC_HOOK_DEBUG=1 时打 stack。让用户看 stderr 就知道某 hook 在 allow-on-throw。

### Bootstrap 加固（1 条）

`bootstrap.sh`: 
- `MCC_DOTFILES_REPO` 协议白名单（https / git@ / ssh），拒绝 `git://` 和本地路径
- JSON config 写入改走 `python3 -c json.dumps(...)`，heredoc 字符串拼接的注入风险消除
- 没 python3 时含特殊字符（"/\\/换行）直接 fail

## 没修的（留 v2.5.13+）

- `installer.js` 全同步 fs → async/await 改造（约 ~1000 行重构）
- `lib/utils.js` `runCommand` execSync → spawnSync 改造（行为兼容性需谨慎）
- 复杂逻辑补单元测试（`mergeSettingsJson` / `detectGitCommand` / `appendTomlFragment`）
- `gateguard pruneStaleFiles` 同步 IIFE → async + 抽样
