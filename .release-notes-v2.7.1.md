## 🔴 HOTFIX: v2.7.0 rate-limit 假阳性

### Bug

```
codex audit 一段含 "rate-limit" 主题的代码（比如 v2.7.0 自己的 codex-runner.js）
  ↓
codex 输出里自然提到 "rate-limit" / "exceeded" / "quota" 等词（讨论代码主题）
  ↓
v2.7.0 runner 把 stderr + stdout 合并扫这 7 个关键词
  ↓
误判为 "codex 限流" → 写 flag → skipped 不阻塞主线
  ↓
用户实测："我刚跑 codex 没限流，你为什么要等"
```

经典 LLM 自指 bug — runner 在审 codex 输出讨论 codex 自身限流机制时，把讨论当报错。

### Fix

```js
// v2.7.0 (有 bug):
const combined = stderr + '\n' + stdout;
if (RATE_LIMIT_PATTERNS.some(re => re.test(combined))) {
  // 误判 stdout 含关键字时
}

// v2.7.1 (修复):
const isRealRateLimit = (result.error || result.status !== 0)  // 必须 exit 非 0
                       && RATE_LIMIT_PATTERNS.some(re => re.test(stderr));  // stderr only
if (isRealRateLimit) {
  // 双校验：exit 非 0 + stderr 含关键词，才算真限流
}
```

**核心规则**：
- `stdout` **永不扫** rate-limit 关键词（codex 的正常输出可能讨论这些主题）
- `stderr` 才看（codex 真错时输出到这里）
- `exit code` 必须非 0（exit 0 = 成功，无论输出含啥词）

测试更新：新 case `stdout 含 rate-limit 关键字 + exit 0 → 不误判`。

## 🆕 Layer 2 二审 + auto-fix

### 三层架构

```
Layer 1: codex (OpenAI)        提 finding
   ↓ runCodexAudit()
   ↓ parseFindings()
   ↓ [{severity, file, line, summary, raw}]
Layer 2: finding-validator     fresh Claude subagent，无原对话 context 独立复现
   ↓ verdict: confirmed | rejected | ambiguous
Layer 3: 主 Claude              auto-decide
   ✓ confirmed   → Edit 修，不询问用户
   ✗ rejected    → 记 docs/adr/codex-rejection-{date}.md
   ? ambiguous   → 升给用户拍板（少数情况）
```

### 关键安全约束

**Layer 2 严格判别——错给 confirmed 比错给 ambiguous 危害大**（confirmed 等于授权 Layer 3 自动修）。

`finding-validator` agent prompt 写明：
- 必须 Read/Grep/Bash 复现，不能盲信 codex 文本
- 攻击向量必须能实测触发或具体步骤说明
- 给 confirmed 必须提供 `FIX_GUIDANCE`（unified diff 或 implementation steps）
- **"宁可 ambiguous 也不要错 confirmed"**

### 二审通过自动修流程

```js
// Phase 3: 派 fresh subagent 二审（一条 message 多 Task call 真并行）
const validations = await Promise.all(findings.map(f =>
  Task({
    subagent_type: 'finding-validator',
    prompt: buildValidatorPrompt(f, gitRange, projectRoot),
  })
));

// Phase 4: 按 verdict 自动分流
for (let i = 0; i < findings.length; i++) {
  const v = parseVerdict(validations[i]);
  if (v.verdict === 'confirmed') {
    applyFix(findings[i], v.fix_guidance);  // 自动 Edit，不问用户
  } else if (v.verdict === 'rejected') {
    appendToADR(findings[i], v);
  } else {
    ambiguous.push({findings[i], v});  // 升给用户
  }
}
```

// v2.7.0:  spawnSync(bin, args, { shell: true })
