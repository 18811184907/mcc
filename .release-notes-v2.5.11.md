# MCC v2.5.11 · 安全审计修复（10 个 P0 Critical）

## 起源：4 agent 并行审计

派了 `security-reviewer` / `code-reviewer` / `silent-failure-hunter` / `javascript-pro` 四个 agent 并行扫 MCC 代码库（约 9300 行 JS，installer + adapters + 28 个 hook 脚本）。

四份独立报告交叉验证后，10 条**多 agent 同时标红**的 Critical 问题被定为本次必修：4 个安全漏洞、3 个数据安全/正确性 bug、3 个静默失败洞。

## 修复清单

### 🔴 安全（4 条）

| # | 文件 | 风险 |
|---|---|---|
| C-1 | `source/hooks/scripts/hooks/post-claudemd-sync.js` | `config.syncFile` 未校验 → `../../.ssh/authorized_keys` 路径穿越覆写 SSH 公钥 |
| C-2 | 同上 | 自动 `git push` 不校验远程 URL → 攻击者能改 config 把全局 CLAUDE.md 持续泄漏到自己仓库 |
| C-3 | `source/hooks/scripts/hooks/post-vault-sync.js` | SSH 字段未净化换行 → `~/.ssh/config` 注入 `ProxyJump`、任意 SSH 指令 |
| C-4 | `source/hooks/scripts/hooks/auto-tmux-dev.js` | Windows `cmd /k "..."` 包装时双引号转义不挡 `&` `\|` `^` 等元字符 → 命令注入 |

**修法**：白名单正则校验 + `path.resolve` 边界检查 + `git remote get-url` 实际 URL 比对 config + 严格命令模式校验（含元字符直接 pass-through 不重写）。

### 🟠 数据安全 / 正确性（3 条）

| # | 文件 | 问题 |
|---|---|---|
| CRITICAL-1 | `scripts/installer.js:767` | Codex exclusive 备份的 `catch{}` 吞所有错误，跨盘符 copy 中途失败时 `rmSync` 仍把源目录删空 = 数据丢失 |
| CRITICAL-3 | `scripts/uninstaller.js:212` | Codex 卸载的 `targetRoot` 计算错误，全局 scope 落到 `$HOME` —— 当前 dead code 没触发，但是埋好的删 `~/AGENTS.md` 定时炸弹 |
| MAJOR-2 | `scripts/installer.js` | `extractTomlSections` 把 TOML 注释行里的 `[section]` 当真 section header → 后续真定义被识为重复，MCP 配置静默丢失 |

**修法**：跟 Claude Code 侧逻辑对齐（按 EXDEV/EPERM 过滤 catch、`backupDirAtomicCrossDevice` 替代 copy+rm）+ 按 scope 推导 targetRoot + TOML 跳过注释行。

### 🟡 静默失败（3 条）

| # | 文件 | 用户感知 |
|---|---|---|
| Silent-C1 | `pre-vault-leak-detect.js` | 扫描器抛错被吞 → 真 secret 静悄悄发出去，hook 透明化失效 |
| Silent-C2 | `gateguard-fact-force.js` | `saveState` 写失败被吞 → gate 状态随机化，"有时拦有时不拦" |
| Silent-C3/4 | `session-{start,end}.js` | 顶层 `.catch` 只打 `err.message` 丢 stack → "session 记忆怎么丢了"找不到原因 |

**修法**：所有 swallow-and-continue 改为 `process.stderr.write` 完整 stack。设计上仍 `exit(0)` 不阻断 Claude，但 stderr 留诊断线索。

## 升级建议

如果你装了 v2.5.10 或更早，**强烈建议升级**：

```bash
cd <wherever-you-cloned-mcc>
git pull
node scripts/installer.js   # 默认 smart 模式重装
```

C-1 / C-2 这两条对**全局装了 MCC + 用 /claudemd-sync 同步全局 CLAUDE.md** 的用户暴露最大；C-3 对**用 PROJECT_VAULT.md + SSH section** 的项目暴露；其他几条是数据安全和体验诊断改善。
