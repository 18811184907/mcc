## 你要做的全部事

### 1️⃣ 装（一次性，3 分钟）

管理员会发给你一个文件：`team-install.ps1`

**右键点这个文件 → 选"用 PowerShell 运行"**（不是双击，要**右键**）

⚠ 如果报错 "脚本被禁用"：
> 打开 PowerShell（开始菜单搜 "PowerShell"），复制下面这行回车：
>
> ```powershell
> Set-ExecutionPolicy -Scope CurrentUser RemoteSigned -Force
> ```
>
> 然后再右键点脚本运行一次。

看到 **"全部完成 🎉"** 就 OK 了。

### 2️⃣ 用（每天写完代码，10 秒）

打开 Claude Code，在你项目里敲：

```
/backup "简短说明你改了啥"
```

例子：
- `/backup "修了首页 bug"`
- `/backup "加了导出按钮"`
- `/backup "改完设计稿"`

完事。

**第一次**用时会问你两个问题（姓名 + 邮箱），填完就好，以后不问了。

## 出状况了怎么办

### Q1: 脚本运行一半报错

→ **截图发给管理员**，他会看是哪一步挂了

### Q2: 敲 `/backup` 说"推送失败"

大概率是：
- 🔴 **断网了** → 连上网再试一次
- 🔴 **管理员的 PAT 过期了** → 找管理员要新脚本，再运行一次 `team-install.ps1`

### Q3: 推送很慢

正常。项目第一次推要把所有代码传上去，几十兆要 10-60 秒。之后每次只传差异，通常 1-3 秒。

### Q4: 改错了想撤回上次 `/backup`

在 Claude Code 里说："帮我撤回刚才那次 backup" — Claude 会告诉你怎么做。

### Q5: 我想关掉某个项目的备份

敲 `/backup-off`

---

## 重要提醒

- 🔒 **别把 `team-install.ps1` 发给外部人**（里面有公司密钥）
- 🔒 **不要在个人项目里用 `/backup`**（这是公司工具，别把私人代码备份到公司仓库）
- 📧 **3 个月管理员可能会给你新脚本**（PAT 到期要换），重新运行一次就好

## 快速卡

```
/backup "说明"      ← 每天用这个
/backup-status     ← 看状态
/backup-off        ← 不想备份某项目了
```

三条命令，日常只用第一条。
