## 🔴 CRITICAL: 所有 hook 都没在工作

**真相揭露**：v2.5.x 起 ${MCC_HOOKS} 替换 bug 一直存在，没人发现因为 hook 静默失败 + 测试通过（vault-hook-test 等通过 spawn source 文件绕过 settings.json）。

```
hooks.json 自描述 (line 8)：
  "MCC_HOOKS": "${MCC_HOME}/hooks/scripts"      ← 应该指向 .mcc-hooks/scripts

installer.js line 649 实际：
  MCC_HOOKS: mccHome                             ← 替成 .mcc-hooks (漏 'scripts/')

部署 settings.json (实测):
  "node /Users/.../.mcc-hooks/hooks/session-start-bootstrap.js"   ← 不存在
                       ↑↑↑↑↑↑↑↑
                       缺 'scripts/'

实际文件位置:
  /Users/.../.mcc-hooks/scripts/hooks/session-start-bootstrap.js  ← 真路径
```

**影响所有 hook**：session-start-bootstrap, session-end, check-console-log, pre-vault-leak-detect, post-vault-sync, post-claudemd-sync, post-user-vault-sync — 全部静默失败。

**Fix**：installer.js 两处 (line 644 + 697) 把 MCC_HOOKS 替换值改成 `path.join(mccHome, 'scripts')`，跟 hooks.json 自描述对齐。

## 🟠 HIGH: session-start-bootstrap 找不到 MCC

`resolvePluginRoot()` 只查 `ecc` / `everything-claude-code` 名字 + ~/.claude direct，**完全不知道有 `.mcc-hooks` 目录**。fallback 到 ~/.claude → 查 `~/.claude/scripts/hooks/run-with-flags.js`（不存在）→ hook 静默 skip → session 记忆/上下文注入失效。

**Fix**：加 `~/.claude/.mcc-hooks` 作为优先级 2 候选（env override 后立即查），其他逻辑不动。

## 🟡 MEDIUM: smoke.js 23 失败 → 0 失败

```
parser bug:    body 末行孤儿 \r → 22 SKILL.md 误报
README check:  动态 badge 改了 → 1 README 误报
```

**Fix**：parser 先 `.replace(/\r/g, '')` 剥光 \r；README 检查接受动态 `img.shields.io/github/v/tag/...` URL。

**Result**: smoke 23 fail → **206 checks 全过 0 失败**。

## 🟡 MEDIUM: 加 Case 10 真装验证

```
Before: dry-run 不写文件 → 漏掉 ${MCC_HOOKS} 替换 bug
After:  Case 10 真装到 tmp + 解析 settings.json + 逐条 fs.existsSync 验证 hook command
```

将来 hook 路径类 bug 不会再溜过测试。22 checks (19 → 22)。

## ⚠️ REJECTED: PROJECT_VAULT 相对路径正则

codex 字面观察：`/\/docs\/PROJECT_VAULT\.md$/` 前导 `/` 漏相对路径 `docs/PROJECT_VAULT.md`。

**Validator verdict: rejected**。理由：
- Claude Code Write/Edit tool_input.file_path 始终绝对路径（系统行为）
- 同仓库 6 个 hook 都直接用 file_path 不 normalize，整体共识
- Windows 绝对路径 `C:/...` 也命中前导 `/`
- 退一步 fs.existsSync 兜底 no-op 不会错误同步

记 `docs/adr/codex-rejection-2026-04-30.md` 含完整 reasoning。
